# Common Mistakes

- Thinking XML is only for websites. XML is mostly used for structured data, not page layout.
- Expecting XML to allow multiple root elements. A normal XML document must have one root element.
- Treating XML tags as case-insensitive. `<Book>` and `<book>` are different.
- Assuming attributes and elements are interchangeable in every case. They serve different design purposes.
- Skipping the repository order and jumping into XQuery before learning XPath and validation.
