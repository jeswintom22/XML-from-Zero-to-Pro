# XPath and XQuery Cheat Sheet

- `/a/b` selects a direct path
- `//b` selects matching nodes anywhere
- `@id` selects an attribute
- `[price > 50]` filters with a predicate
- `count(...)` counts nodes
- XQuery FLWOR: `for`, `let`, `where`, `order by`, `return`
- XQuery can build new XML output, not just select existing nodes
